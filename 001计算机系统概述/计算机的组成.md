## 2. 冯诺依曼机

世界上第一台计算机ENIAC需要程序员手动接线来控制计算, 虽然它的计算速度很快, 但是被手工操作的耗时抵消掉了.

于是 冯诺依曼第一次提出了**存储程序**的概念.所谓存储程序是指将指令以二进制代码的形式事先输入到计算机的主存储器, 然后按其在存储器中的首地址执行程序第一条指令, 以后就按照规定顺序执行其它指令, 直到程序结束.



 <p align="center"><img src="./assets/image-20260624152201858.png" width="45%" /></p>



图中实线是数据线, 虚线是反馈线.

冯诺依曼机特点:

- 计算机由五大部件组成.分别是运算器、控制器、存储器、输入设备和输出设备.
  - 主存储器: 存放数据和程序
  - 运算器: 用来算术运算(加减乘除)和逻辑运算(与或非)
  - 控制器: 指挥各部件, 使程序运行.
  - 输入设备: 鼠标、键盘、扫描仪等等
  - 输出设备: 显示器、打印机等等
- 指令和数据以同等地位存于存储器, 可按地址寻访.
- 指令和数据都是以二进制表示.
- 指令由**操作码**和**地址码**组成.
- 引入了存储程序.
- 以运算器为中心.





**补充说明**

上面说到冯诺依曼机是以运算器为中心， 虽然有了存储程序, 但是存储程序仍然要通过运算器写入到存储器里面.

CPU = 运算器 + 控制器.

## 3. 计算机硬件



### 3.1 主存储器

<p align="center"><img src="./assets/image-20260624162625258.png" width="45%" /></p>



MAR: Memory Address Register 存储地址寄存器

MDR: Memory Data Register 存储数据寄存器

- 存储体是存放实际数据的地方, 一个程序的很多数据.
- MAR存放访存地址，经过地址译码后找到所选的存储单元.
- MDR用于暂存从存储体中要读或写的数据.

示例: CPU需要读取一个int类型数据

- 先将数据的地址写入MAR, 假设是 0x00400000.
- 在存储体中找到 0x00400000位置, 读取4字节数据, 写入到MDR中.
- CPU从MDR中读取数据.





关于存储体:

- 由连续的存储单元组成.下标就是地址,从0开始.
- 存储字长: 指的是存储单元里面可以存储多少个二进制位, 一般是 8bit, 16bit， 32bit等等.
- 如果MAR是4位, MDR是16位
  - 一共有 2<sup>4</sup> = 16个存储单元
  - 每个存储单元可以存放16bit信息
  - 一共可以存储 16 * 16 = 256 bit 数据.

### 3.2 运算器

运算器的作用是实现算数运算和逻辑运算， 即加减乘除和与或非.

<p align="center"><img src="./assets/image-20260624162423638.png" width="20%" /></p>

ACC: Accumulator,累加器,用于存放操作数或运算结果.

MQ: Multiple-Quotient Register, 乘商寄存器, 在乘除运算时, 用于存放操作数或运算结果.

ALU: Arithmetic and Logic Unit, 算数逻辑单元, 通过内部复杂电路实现算术运算和逻辑运算.

X: 通用操作数寄存器, 用于存放操作数. 



|      | 加         | 减         | 乘             | 除           |
| ---- | ---------- | ---------- | -------------- | ------------ |
| ACC  | 被加数、和 | 被减数、差 | 乘积高位       | 被除数、余数 |
| MQ   |            |            | 乘数. 乘积低位 | 商           |
| X    | 加数       | 减数       | 被乘数         | 除数         |



### 3.3 控制器



<p align="center"><img src="./assets/image-20260624165048786.png" width="25%" /></p>

- CU: Control Unit, 控制单元,分析指令, 给出控制信号 (CU是控制器的核心, 里面有非常复杂的电路)
- IR：Instruction Register, 指令寄存器, 存放当前指令的指令
- PC: Program Counter, 程序计数器





- PC用来存放当前要执行的指令的地址, 且具有自动加 **1** 功能, 这里的 **1** 是指一条指令的长度, 即可以自动形成下一条指令的地址, 它和主存储之间有一条直接通路.

- IR用来存放当前指令, 其内容主要来自主存储器的MDR, 指令中的操作码OP(IR)被送到CU, 而地址码Ad(IR)被送到MAR，用来取操作数.

## 5. 计算机的工作过程



以下面程序为例子:

```cpp
int a = 2, b = 3, c = 1, y = 0;
void main()
{
    y = a * b + c;
}

```



<img src="./assets/image-20260625120855800.png" alt="image-20260625120855800" style="zoom: 50%;" />

#初: (PC) = 0, 指向第一条指令地址

#1: (PC)-->MAR, 导致(MAR) = 0

#3: M(MAR)-->MDR, 导致(MDR) = 000001 0000000101

#4: (MDR)->IR, 导致(IR) = 000001 0000000101

#5: OP(IR)-->CU, 指令的操作码被送到CU, CU分析指令后得知, 这是**取数**指令， 于是CU把 0000000101 放到MAR中,换算成10进制就是5 

#6: Ad(IR)-->MAR, 指令的地址码送到MAR, 导致(MAR)=5.

#8: M(MAR)-->MDR, 导致(MDR) = 0x0000 0000 0000 0010  = 2.   从存储体地址5中取出数据放到MDR中.

#9: (MDR)--> ACC, 导致(ACC)= 0x0000 0000 0000 0010 = 2.



上面执行完之后, (PC)=1，(ACC)=2. 总结上面的操作

- #1 ~ #4, 取指令
- #5, 分析指令
- #6 ~ #9, 执行指令



<img src="./assets/image-20260625154555882.png" alt="image-20260625154555882" style="zoom: 50%;" />



#1: (PC)-->MAR, 导致 (MAR) = 1.

#3: M(MAR)-->MDR, 导致 (MDR) = 000100 0000000110

#4: (MDR)-->IR, 导致 (IR) = 000100 0000000110

#5: OP(IR)送到CU, 即指令的操作码被送到CU, CU分析指令后得知, 这是**乘法指令**. 需要取数到MQ中

#6: Ad(IR)-->MAR, 执行完后, (MAR) = 0x0000000110. 换算成十进制就是6.

#8. M(MAR)--> MDR, 导致 (MDR) = 0x0000 0000 0000 0011, 即 (MDR) = 3.

#9. (MDR) --> MQ, MDR中的数据通过数据总线送到MQ中, 导致 (MQ) = 3.

#10. (ACC) --> X, 导致(X) = 2.  因为是 X 寄存器里面可以存放被乘数.

#11. (MQ)* (X) --> ACC, 由ALU实现乘法运算, 导致 (ACC) = 6, 如果乘积太大, 则需要MQ辅助存储.MQ会保存乘积低位, ACC来保存乘积高位.



上面执行完后, (PC)=2, (ACC)=6, 总结上面操作:

- #1 ~ #4 取指令
- #5 分析指令
- #6 ~ #11 执行乘法指令
